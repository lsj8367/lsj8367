숨김파일 표시하면 .git 생성되어있는 폴더에
최초 방법====
git init
git remote add origin [ repository 주소 ]

git clone https://github.com/lsj8367/test.git
git remote origin 링크
git config --global user.name 이름
git config --global user.email 이메일
git remote add origin {remote repository 주소}
모두올리기 - git add * 또는 git add .
설정하고
========
만들어진 폴더에 git status하면 만든 파일들 나옴(빨간글씨)
git add 파일명
다시 git status 하면 초록글씨로 commit 가능한상태
여기서 잘못 올렸다면
git restore --staged 파일명 으로 add 취소


git commit -m "하고싶은 커밋 문구"해서 commit

로컬저장소와 깃허브 원격저장소(온라인)를 연결합니다.
git remote add origin 자기주소
git remote -v : 연결 확인

마지막 : git push 하고 깃허브 계정 로그인
여기서 오류 발생할 시에는 git push origin master로 실행할것.


수정된거 내려받기
git pull --rebase { 원격 저장소 별명 } { 브랜치명 }  origin master

git branch -a 별칭확인
git checkout [브랜치 이름]  브랜치 변경
git branch [브랜치명]      브랜치 생성
git branch -d 브랜치명

원격 저장소 삭제
git rm --cached Practice.java 
git commit -m "remove Practice.java" 
git push

에러 뜰때
LF어쩌구 에러
git config --global core.autocrlf true

로컬저장소 안쓰는파일 삭제해버리는거
git rm 파일명

원격 저장소에서 삭제하는법
폴더는 git rm -r 하고
git rm => 원격 저장소와 로컬 저장소에 있는 파일을 삭제
git rm –cached => 로컬 저장소에 있는 파일은 삭제하지 않고 원격 저장소에 있는 파일만 삭제
git commit -am 'myFile 삭제!!'
하고 git push


[Git] ! [rejected] master -> master (non-fast-forward) 해결 방법
해결 방법은 push하려고하는 브랜치 이름 앞에 + 를 붙여 push 하면 됩니다.
git
 $ git push origin +master 

.gitignore 파일을 만들고 해당 폴더 제외시키고 싶으면
폴더/ 해주고 커밋하면 폴더 제외됨

.gitattributes에는 속성값을 줄수가 있다.

gitignore 안쓰고 무시하는법
$ git update-index --assume-unchanged [파일명]

branch 협업
branch 변경 git checkout 브랜치명
원격 저장소를 만들고 나머지 멤버 git clone URL주소
git branch 생성해서 나눔
서로 다른 작업을 수행하는데 작업이 완료되었다면,
git add '파일명'
git commit -m "수정(입력)사항 입력"

git checkout master << 마스터브랜치로 이동
git merge 자신의 브랜치명 << master브랜치와 병합
merge 후에 새로운 파일이 master브랜치에 있을경우
git pull origin master 해서 최신것을 받아온다.
